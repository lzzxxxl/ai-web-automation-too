# AI网页批量处理助手 - 浏览器插件开发需求文档

## 一、项目概述

### 1.1 项目名称
**AI网页批量处理助手** (AI Web Batch Assistant)

### 1.2 核心功能
一个浏览器插件，用于批量向网页版AI平台（如ChatGPT、DeepSeek、Claude、Grok等）提交问题，自动监控AI回复，将回复内容转换为HTML格式并添加元数据后保存到本地文件。

### 1.3 目标用户
- 需要批量向AI平台提问并保存回复的用户
- 追求高效、无人值守操作的研究人员和内容创作者
- 需要系统化管理AI交互流程的专业用户

### 1.4 价值主张
- 点击安装即可使用，无需配置开发环境
- 直接在浏览器内完成全部操作
- 原生浏览器API，支持Chrome、Edge、Firefox等多种浏览器
- 自动标题提取、元数据处理、Markdown转HTML
- 易于安装和分发给其他用户

## 四、内容转换与元数据处理（核心功能）

### 4.1 标题识别格式

程序支持识别以下格式的标题：

| 格式 | 示例 |
|------|------|
| `**原文章标题：**` | `**原文章标题：** 如何学习Python` |
| `**原文件标题：**` | `**原文件标题：** 技术文档.txt` |
| `**文章标题：**` | `**文章标题：** 文章标题` |
| `**文件标题：**` | `**文件标题：** 文件标题` |
| `原文章标题：` | `原文章标题：如何学习Python` |
| `原文件标题：` | `原文件标题：技术文档.txt` |
| `文章标题：` | `文章标题：文章标题` |
| `文件标题：` | `文件标题：文件标题` |
| `title:` | `title: Article Title` |
| `<title>` | `<title>页面标题</title>` |

### 4.2 自定义匹配字段

用户可在设置中填写自定义字段名（多个用逗号分隔），程序会自动兼容以下三种格式：

| 输入格式 | 匹配示例 |
|---|---|
| Markdown加粗 | `**标题：** 内容` |
| 普通中文冒号 | `标题：内容` |
| 普通英文冒号 | `标题:内容` |
| HTML格式 | `<strong>标题：</strong>内容` |

例如：输入 `标题` 或 `标题：`，可匹配以上所有格式。

### 4.3 WordPress 格式处理

程序支持处理特定格式的剪贴板内容：

**输入格式**：
```
[intro部分 - 将被删除]

***

[文章正文 - 将转换为HTML]

***

**标题：** 文章标题
**SEO描述：** SEO描述内容
**别名简短URL：** article-slug
**WordPress标签：** 标签1,标签2
```

**输出格式**：
```html
<!-- 标题：文章标题 -->
<!-- SEO描述：SEO描述内容 -->
<!-- 别名简短URL：article-slug -->
<!-- WordPress标签：标签1,标签2 -->

<p>文章正文已转换为HTML格式...</p>
```

**处理规则**：
- `***` 或 `---` 之前的 intro 部分会被删除
- 第一个 `***`/`---` 之间的内容作为文章正文，转换为 HTML
- 第二个 `***`/`---` 之间的内容作为元数据，自动提取字段并添加 HTML 注释
- 分隔符之后的元数据行会被解析并转换为 `<!-- 字段名：值 -->` 格式

### 4.4 元数据格式规范

最终保存的文件元数据格式：

```html
<!-- platform: deepseek -->
<!-- title: 什么是Python -->
<!-- timestamp: 2026-04-15 10:30:25 -->
<!-- task_id: 001 -->
<!-- source_url: https://chat.deepseek.com/... -->
```

支持的元数据字段：
- `platform`: 来源AI平台
- `title`: 文章标题
- `timestamp`: 处理时间
- `task_id`: 任务序号
- `source_url`: 来源URL
- 以及用户通过WordPress格式添加的自定义字段

### 4.5 Markdown 转 HTML

支持的Markdown语法：
- 标题 (`#`, `##`, `###`)
- 粗体 (`**text**`)
- 斜体 (`*text*`)
- 代码块 (``` ```)
- 行内代码 (`` ` ``)
- 链接 (`[text](url)`)
- 列表 (`-`, `1.`)
- 引用 (`>`)
- 表格

### 4.6 文件名清洗规则

标题用于文件名匹配时，只保留中文、英文、数字：
- 输入：`**文章标题：** Python入门教程-v2.0`
- 清洗后：`Python入门教程v20`

## 五、自动化流程设计

### 5.1 主流程
```
1. 用户添加标题 → 任务入队
2. 取出一个任务 → 状态变为"处理中"
3. 切换到AI平台标签页
4. 检测AI平台类型
5. 定位输入框 → 填写标题
6. 点击发送按钮
7. 进入监控循环:
   - 检测AI回复状态
   - 如果有"继续生成"按钮 → 点击 → 继续监控
   - 如果回复完成 → 点击复制 → 进入步骤8
8. 内容处理:
   - 获取剪贴板内容
   - 提取标题（如有）
   - 处理WordPress格式（如有***分隔符）
   - 转换为HTML
   - 添加元数据注释
9. 保存文件:
   - 生成文件名（序号_清洗后标题_时间戳）
   - 按配置创建文件夹
   - 写入内容
10. 任务状态变为"已完成"
11. 取下一个任务 → 重复步骤2-10
```

### 5.2 内容转换流程
```
复制的AI回复
    ↓
检查是否包含 *** 或 --- 分隔符
    ↓
┌─────────────────────────────────────┐
│ 是：WordPress格式处理                │
│   - 删除intro部分                    │
│   - 提取正文和元数据                 │
│   - 正文转为HTML                     │
│   - 元数据转为<!-- -->注释           │
├─────────────────────────────────────┤
│ 否：标准处理                         │
│   - 检查是否Markdown格式             │
│   - 转换为HTML（如需要）             │
│   - 保留原格式                       │
└─────────────────────────────────────┘
    ↓
添加平台元数据（platform, timestamp等）
    ↓
生成最终文件内容
    ↓
保存到本地文件
```

### 5.3 状态机
```
IDLE → RUNNING → PAUSED
  ↓       ↓         ↓
  └───────┴──→ COMPLETED
              ↓
            FAILED
```

### 5.4 异常处理
- 输入框未找到：等待2秒后重试，最多3次
- 发送按钮未找到：记录错误，跳过任务
- AI回复超时：自动点击继续生成或跳过
- 复制失败：尝试其他复制方式（Ctrl+C、右键菜单）
- 内容转换失败：保存原始文本内容
- 网络错误：暂停任务，等待恢复后继续

- 检测当前浏览器类型
- 提供统一的API抽象
- 处理不同浏览器的API差异

#### 9.2.8 BackgroundService (后台服务)

## 十、配置文件格式

### 10.1 插件配置 (watcher_config.json)

```json
{
  "last_folder": "C:/Users/xxx/Documents/AI回复",
  "auto_save": true,
  "always_on_top": false,
  "theme": "system",
  "markdown_convert": true,
  "match_patterns": "**原文章标题**,**原文件标题**,**文章标题**,**文件标题**",
  "platforms": ["deepseek", "chatgpt", "claude"],
  "save_mode": "new_file",
  "folder_mode": "by_platform"
}
```

| 配置项 | 类型 | 说明 |
|--------|------|------|
| `last_folder` | string | 上次使用的保存路径 |
| `auto_save` | boolean | 是否自动保存 |
| `always_on_top` | boolean | 窗口是否置顶 |
| `theme` | string | 主题：dark、light、system |
| `markdown_convert` | boolean | 是否启用Markdown转HTML |
| `match_patterns` | string | 自定义标题匹配字段，逗号分隔 |
| `platforms` | array | 启用的AI平台列表 |
| `save_mode` | string | 保存模式：new_file、append |
| `folder_mode` | string | 文件夹分类：by_platform、by_date、none |
