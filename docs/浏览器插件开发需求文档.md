# AI网页批量处理助手 - 浏览器插件开发需求文档

## 一、项目概述

### 1.1 项目名称
**AI网页批量处理助手** (AI Web Batch Assistant)

### 1.2 核心功能
一个浏览器插件，用于批量向网页版AI平台（如ChatGPT、DeepSeek、Claude、Grok等）提交问题，自动监控AI回复，并在回复完成后自动保存结果。

### 1.3 目标用户
- 需要批量向AI平台提问并保存回复的用户
- 追求高效、无人值守操作的研究人员和内容创作者
- 需要系统化管理AI交互流程的专业用户

### 1.4 价值主张
- 将现有的Python桌面应用功能迁移到浏览器插件
- 无需额外部署Python环境和依赖
- 直接在浏览器内完成全部操作，更便捷
- 利用浏览器原生API实现更稳定的浏览器控制

## 二、功能需求

### 2.1 核心功能模块

#### 2.1.1 任务队列管理
- **FR-1.1**: 支持从TXT文件导入标题列表
- **FR-1.2**: 支持手动粘贴/输入标题（每行一个）
- **FR-1.3**: 自动去重功能，避免重复处理
- **FR-1.4**: 任务状态跟踪（待处理、处理中、已完成、失败）
- **FR-1.5**: 支持暂停、继续、跳过任务
- **FR-1.6**: 任务进度实时显示

#### 2.1.2 AI平台交互
- **FR-2.1**: 自动检测当前网页的AI平台类型
  - 支持ChatGPT (chat.openai.com)
  - 支持DeepSeek (chat.deepseek.com)
  - 支持Claude (claude.ai)
  - 支持Grok (grok.x.ai)
  - 支持Perplexity (perplexity.ai)
  - 支持Coze (coze.com)
- **FR-2.2**: 自动定位并填充输入框
- **FR-2.3**: 自动检测并点击发送按钮
- **FR-2.4**: 智能检测"继续生成"按钮并自动点击
- **FR-2.5**: 监控AI回复状态（生成中、等待、完成）
- **FR-2.6**: 回复稳定后自动点击复制按钮
- **FR-2.7**: 支持配置最大等待时间和重试次数

#### 2.1.3 结果保存
- **FR-3.1**: 自动保存AI回复到本地文件
- **FR-3.2**: 文件命名规则：`{序号}_{标题}_{时间戳}.txt`
- **FR-3.3**: 支持自定义保存路径
- **FR-3.4**: 自动创建以标题命名的文件夹分类保存
- **FR-3.5**: 支持多种导出格式（纯文本、Markdown、JSON）
- **FR-3.6**: 保存记录日志，便于追溯

#### 2.1.4 浏览器控制
- **FR-4.1**: 连接已打开的浏览器标签页
- **FR-4.2**: 在多个AI平台标签页间切换
- **FR-4.3**: 自动刷新/重新加载页面
- **FR-4.4**: 登录状态保持检测

### 2.2 用户界面需求

#### 2.2.1 插件popup界面
- 简洁的弹出面板，显示任务列表和进度
- 一键开始/暂停/停止功能
- 当前任务状态显示
- 快速添加标题入口

#### 2.2.2 侧边面板（Side Panel）
- 完整的任务管理界面
- 实时日志显示
- 配置选项
- 统计信息（处理数量、成功率等）

#### 2.2.3 设置页面
- 保存路径配置
- AI平台选择
- 自动化参数配置
- 导出/导入配置

### 2.3 数据存储需求
- **DR-1**: 使用Chrome Storage API存储配置
- **DR-2**: 使用IndexedDB存储任务队列和历史记录
- **DR-3**: 支持导出/导入配置和数据

## 四、自动化流程设计

### 4.1 主流程
```
1. 用户添加标题 → 任务入队
2. 取出一个任务 → 状态变为"处理中"
3. 切换到AI平台标签页
4. 检测AI平台类型
5. 定位输入框 → 填写标题
6. 点击发送按钮
7. 进入监控循环:
   - 检测AI回复状态
   - 如果有"继续生成"按钮 → 点击 → 继续监控
   - 如果回复完成 → 点击复制 → 保存结果
8. 任务状态变为"已完成"
9. 取下一个任务 → 重复步骤2-8
```

### 4.2 状态机
```
IDLE → RUNNING → PAUSED
  ↓       ↓         ↓
  └───────┴──→ COMPLETED
              ↓
            FAILED
```

### 4.3 异常处理
- 输入框未找到：等待2秒后重试，最多3次
- 发送按钮未找到：记录错误，跳过任务
- AI回复超时：自动点击继续生成或跳过
- 复制失败：尝试其他复制方式（Ctrl+C、右键菜单）
- 网络错误：暂停任务，等待恢复后继续

## 六、用户操作场景

### 6.1 场景一：批量提问DeepSeek
1. 用户在DeepSeek网站登录
2. 打开插件侧边栏
3. 粘贴100个问题到任务列表
4. 点击"开始处理"
5. 插件自动逐个处理每个问题
6. 每个问题回复自动保存到本地文件
7. 用户可随时查看进度或暂停

### 6.2 场景二：多平台切换
1. 用户在多个标签页打开不同的AI平台
2. 在插件中配置多个任务队列
3. 插件自动在不同平台间切换处理
4. 统一管理所有平台的回复保存

### 6.3 场景三：中途暂停与恢复
1. 用户开始批量处理
2. 突然需要手动操作电脑
3. 点击"暂停"按钮
4. 插件停止，等待用户
5. 用户操作完成后，点击"继续"
6. 插件从断点处继续执行

## 七、非功能性需求

### 7.1 性能要求
- 页面加载响应时间 < 100ms
- 任务切换时间 < 500ms
- 内存占用 < 100MB
- 不影响浏览器正常浏览

### 7.2 稳定性要求
- 连续运行8小时以上无崩溃
- 任务执行成功率 > 95%
- 自动错误恢复机制
- 数据自动保存

### 7.3 安全性要求
- 不收集用户隐私数据
- 不上传用户问题内容
- 本地存储，数据不外泄
- 遵循Chrome扩展安全规范

### 7.4 可维护性要求
- 模块化代码结构
- 完善的注释和文档
- 易于扩展新平台
- 支持版本更新

## 八、插件结构设计

### 8.1 目录结构
```
ai-web-batch-assistant/
├── manifest.json           # 插件配置文件
├── background.js          # 后台服务脚本
├── content.js             # 内容脚本（注入AI页面）
├── sidepanel.html         # 侧边面板界面
├── sidepanel.js          # 侧边面板逻辑
├── popup.html             # 弹出窗口界面
├── popup.js              # 弹出窗口逻辑
├── settings.html         # 设置页面
├── settings.js          # 设置逻辑
├── styles/
│   ├── common.css        # 公共样式
│   ├── sidepanel.css    # 侧边栏样式
│   └── popup.css        # 弹窗样式
├── lib/
│   ├── ai-platforms/    # AI平台适配器
│   │   ├── chatgpt.js
│   │   ├── deepseek.js
│   │   ├── claude.js
│   │   └── grok.js
│   ├── task-manager.js  # 任务管理器
│   ├── storage.js       # 存储管理
│   └── utils.js         # 工具函数
├── assets/
│   ├── icons/           # 图标资源
│   └── images/          # 图片资源
└── _locales/            # 国际化资源
    ├── en/messages.json
    └── zh_CN/messages.json
```

### 8.2 核心模块说明

#### 8.2.1 TaskManager (任务管理器)
- 管理任务队列的增删改查
- 维护任务状态机
- 提供进度统计接口

#### 8.2.2 AIPlatformAdapter (AI平台适配器)
- 每个AI平台一个适配器类
- 统一接口定义
- 平台特定的选择器和操作逻辑

#### 8.2.3 StorageManager (存储管理器)
- 封装Chrome Storage API
- 数据序列化/反序列化
- 配置和任务数据管理

#### 8.2.4 BackgroundService (后台服务)
- 消息路由
- 定时任务
- 下载管理
